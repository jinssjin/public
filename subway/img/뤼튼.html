코드에서 on 클래스와 active 클래스가 제대로 작동하지 않는 문제를 해결하기 위해 몇 가지 점검할 사항이 있습니다. 아래는 문제를 해결하기 위한 몇 가지 제안입니다.

  1. HTML 구조 확인: #menus와 #sanmenusub의 HTML 구조가 올바른지 확인하세요. #menus는 탭 버튼을 포함하고, #sanmenusub는 각 탭에 해당하는 내용을 포함해야 합니다. 예를 들어:

    </ul>
    <ul>
        <li>Content 2</li>
    </ul>
    <ul>
        <li>Content 3</li>
    </ul>
</div>

  2. CSS 확인: on 클래스와 active 클래스에 대한 CSS 스타일이 정의되어 있는지 확인하세요. 예를 들어, active 클래스가 적용된 요소의 색상을 변경하는 CSS가 필요합니다.

    css
    .active {
    color: red; /* 예시: 활성화된 탭의 색상 */
}

.on {
    display: block; /* 예시: 활성화된 내용 표시 */
}

#sanmenusub > ul {
    display: none; /* 기본적으로 모든 내용 숨기기 */
}

  3. JavaScript 오류 확인: 브라우저의 개발자 도구(콘솔)를 열어 JavaScript 오류가 발생하는지 확인하세요. 오류가 발생하면 해당 오류를 수정해야 합니다.

  4. 이벤트 리스너 확인: menus 배열의 각 요소에 대해 클릭 이벤트가 제대로 등록되고 있는지 확인하세요. forEach 루프가 제대로 작동하는지 확인하기 위해 console.log를 추가해보세요.

    javascript
    menus.forEach(function(tab, i) {
    tab.addEventListener('click', function(e) {
        console.log('Tab clicked:', i); // 클릭한 탭 인덱스 출력
        // 나머지 코드...
    });
});

  5. e.preventDefault() 사용: 만약 탭이 <a> 태그로 구성되어 있다면, e.preventDefault()를 주석 해제하여 링크의 기본 동작을 방지해야 합니다.

이러한 점검 사항을 확인한 후에도 문제가 해결되지 않는다면, 추가적인 코드나 오류 메시지를 제공해 주시면 더 구체적인 도움을 드릴 수 있습니다.

3. JavaScript 코드 점검

JavaScript 코드에서 on 클래스와 active 클래스가 제대로 작동하도록 하기 위해 다음 사항을 확인하세요:

  • e.preventDefault()를 주석 해제하여 링크의 기본 동작을 방지해야 합니다. 만약 탭이 <a> 태그로 구성되어 있다면 이 부분이 필요합니다.
  • sanmenusub의 각 ul 요소가 기본적으로 숨겨져 있어야 하며, on 클래스가 추가된 요소만 표시되어야 합니다.

4. 전체 코드 예시

            });
        });

CSS 코드에서 .slider2 클래스가 서 있는 이유는 여러 가지가 있을 수 있습니다. 주로 레이아웃과 관련된 속성들이 영향을 미칠 수 있습니다. 아래는 .slider2 클래스와 관련된 CSS 속성 및 그 의미에 대한 설명입니다.

1. Flexbox 사용

css
.slider2 {
    display: flex;
    margin-top: 30px;
}
  • display: flex;: 이 속성은 .slider2 요소를 플렉스 컨테이너로 만들어, 자식 요소들이 플렉스 아이템으로 배치됩니다. 플렉스 레이아웃은 요소들을 수평 또는 수직으로 정렬할 수 있게 해줍니다.
  • margin-top: 30px;: 이 속성은 .slider2 요소의 상단에 30픽셀의 여백을 추가하여, 위쪽 요소와의 간격을 만듭니다.

2. 자식 요소의 스타일

css
.slider2 li {
    width: 1920px;
    height: 300px;
    border: 2px solid saddlebrown;
    box-sizing: border-box;
    text-align: center;
}
  • 각 리스트 아이템(li)의 너비가 1920픽셀로 설정되어 있습니다. 이는 화면 전체 너비와 같기 때문에, 여러 개의 슬라이더가 수평으로 나열될 경우, 화면을 넘어갈 수 있습니다.
  • height: 300px;: 각 슬라이더의 높이를 300픽셀로 설정합니다.
  • border: 2px solid saddlebrown;: 슬라이더에 테두리를 추가하여 시각적으로 구분됩니다.
  • box-sizing: border-box;: 이 속성은 패딩과 테두리를 포함하여 요소의 총 너비와 높이를 계산하게 합니다. 따라서, 설정한 너비와 높이가 실제로 보이는 크기와 일치합니다.
  • text-align: center;: 슬라이더 내의 텍스트를 중앙 정렬합니다.

3. 슬라이더의 동작

슬라이더가 서 있는 이유는 주로 다음과 같은 요소들 때문입니다:

  • 너비 설정: 각 슬라이더의 너비가 1920픽셀로 설정되어 있어, 화면에 하나의 슬라이더만 보이게 됩니다. 여러 슬라이더가 있을 경우, 화면을 넘어가게 됩니다.
  • 플렉스 레이아웃: 플렉스 컨테이너로 설정되어 있어, 자식 요소들이 수평으로 정렬됩니다. 만약 여러 개의 슬라이더가 있다면, 이들이 나란히 배치될 것입니다.

4. 해결 방안

슬라이더가 화면에 서 있는 이유가 불필요한 경우, 다음과 같은 방법으로 수정할 수 있습니다:

  • 너비 조정: 슬라이더의 너비를 화면 크기에 맞게 조정하거나, 여러 슬라이더가 나란히 보이도록 설정할 수 있습니다.
  • 슬라이더 갯수 조정: 슬라이더의 갯수를 조정하여, 화면에 맞게 배치할 수 있습니다.
  • CSS 수정: 필요에 따라 CSS 속성을 수정하여 원하는 레이아웃을 구현할 수 있습니다.
